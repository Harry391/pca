# 整体工程拆分与接口说明

## 1. 目标

本文档用于把整个 MATLAB 人脸识别项目拆成一个统一工程，并明确两个人的职责边界、文件边界、接口边界和联调顺序。

目标不是“分别做两堆代码”，而是“做同一个工程中的两个模块”，最终通过一个统一入口运行。

## 3. 两个人的工程边界

### 3.1 A 同学负责的模块

负责目录：

1. `ui/`
2. `preprocess/`
3. `integration/` 中偏界面触发的部分
4. `run_main.m`

负责内容：

1. 主界面搭建
2. 图像预处理页全部交互
3. 摄像头显示与停止启动
4. 结果图、结果表、状态文本刷新
5. 把 B 同学的识别接口接进按钮回调
6. 实时采集人脸送回预处理页复现

### 3.2 B 同学负责的模块

负责目录：

1. `ml/`
2. `integration/` 中偏模型调用的部分
3. `models/` 和 `results/` 的生成逻辑

负责内容：

1. 数据集读取与 2:8 划分
2. 数据集完整性核查
3. 手写 PCA + 手写 SVM 模型训练
4. 单张预测
5. 测试集全量识别
6. 平均脸、特征脸生成
7. 实时帧识别输入输出
8. 评测统计表生成

## 4. 统一状态模型

整个 GUI 运行过程中，建议维护一个统一状态结构体 `appState`。

最低字段如下：

```matlab
appState.currentImage
appState.currentImagePath
appState.currentGrayImage
appState.currentFaceBox
appState.currentAlignedFace
appState.currentCameraFrame
appState.lastReplayPackage
appState.model
appState.batchResult
appState.singleResult
appState.realtimeResult
appState.datasetManifest
appState.defaultPcaDim
appState.defaultSvmC
```

说明：

1. A 同学负责维护 `currentImage`、`currentCameraFrame` 这类界面运行态。
2. B 同学负责定义 `model`、`batchResult`、`singleResult`、`realtimeResult` 的字段结构。
3. `lastReplayPackage` 用于把实时采集结果送回预处理页复现。

## 5. 核心接口定义

下面这些接口必须先冻结，再开始并行开发。

### 5.1 预处理模块输出接口

#### `detect_face`

```matlab
faceInfo = detect_face(img, options)
```

输入：

1. `img`：原始 RGB 或灰度图
2. `options`：检测参数结构体

输出 `faceInfo`：

```matlab
faceInfo.status
faceInfo.faceBox
faceInfo.faceImage
faceInfo.landmarks
faceInfo.message
```

#### `align_face`

```matlab
alignInfo = align_face(img, faceInfo, options)
```

输入：

1. `img`
2. `faceInfo`
3. `options`

输出 `alignInfo`：

```matlab
alignInfo.status
alignInfo.alignedFace
alignInfo.transformInfo
alignInfo.message
```

#### `export_replay_state`

```matlab
replayPkg = export_replay_state(rawFrame, faceInfo, alignInfo)
```

输出 `replayPkg`：

```matlab
replayPkg.rawFrame
replayPkg.faceBox
replayPkg.faceImage
replayPkg.alignedFace
replayPkg.landmarks
replayPkg.timestamp
```

这个结构由 A 输出，供“送入预处理页复现”使用。

### 5.2 模型训练接口

#### `train_pca_svm_model`

```matlab
model = train_pca_svm_model(trainDir, pcaDim, svmC, options)
```

输入：

1. `trainDir`
2. `pcaDim`
3. `svmC`
4. `options`

输出 `model`：

```matlab
model.status
model.message
model.labels
model.imageSize
model.meanFace
model.eigenfaces
model.pcaDim
model.svmC
model.svmParams
model.trainSummary
```

说明：

1. `model.svmParams` 只能保存你们自己手写 SVM 的参数结构。
2. 不允许把 MATLAB 现成分类器对象塞进 `model`。
3. 多分类建议采用你们自己实现的 `one-vs-rest` 或 `one-vs-one` 结构。

### 5.3 单张预测接口

#### `predict_face_identity`

```matlab
result = predict_face_identity(model, imageData, options)
```

输入：

1. `model`
2. `imageData`
3. `options`

输出 `result`：

```matlab
result.status
result.message
result.name
result.topKNames
result.topKScores
result.faceBox
result.alignedFace
result.elapsedMs
```

### 5.4 测试集全量识别接口

#### `run_batch_test`

```matlab
batchResult = run_batch_test(model, testDir, options)
```

输出 `batchResult`：

```matlab
batchResult.status
batchResult.message
batchResult.accuracy
batchResult.totalElapsedMs
batchResult.avgElapsedMs
batchResult.perImageResults
batchResult.confusionSummary
```

其中 `perImageResults` 每行至少包含：

```matlab
trueName
predName
isCorrect
elapsedMs
imagePath
```

### 5.5 平均脸和特征脸接口

#### `build_average_face`

```matlab
avgFace = build_average_face(model)
```

输出：

1. 灰度图矩阵，直接可显示

#### `build_eigenface_preview`

```matlab
eigenfaceImgs = build_eigenface_preview(model, k)
```

输出：

1. `1 x k` 元胞数组
2. 每个元素是一张特征脸图像矩阵

## 6. 界面和算法的对接时机

### 6.1 静态识别按钮

流程：

1. A 读取参数框 `pcaDim`、`svmC`
2. A 调用 B 的 `train_pca_svm_model`
3. A 弹出文件选择框选单张测试图
4. A 调用 B 的 `predict_face_identity`
5. A 把结果显示到识别结果区

### 6.2 测试集全量识别按钮

流程：

1. A 读取参数框
2. A 调用 B 的 `train_pca_svm_model`
3. A 调用 B 的 `run_batch_test`
4. A 用 `update_result_table` 显示逐张结果表
5. A 用文本区显示总体准确率和总耗时

### 6.3 实时识别按钮

流程：

1. A 打开摄像头
2. A 每次取一帧
3. A 调 `detect_face`
4. A 调 `align_face`
5. A 把 `alignedFace` 传给 B 的 `predict_face_identity`
6. A 显示原始图、人脸图、名字、耗时
7. A 同时保存 `replayPkg`

### 6.4 送入预处理页复现按钮

流程：

1. A 读取 `appState.lastReplayPackage`
2. A 把原图、人脸框、人脸图、对齐图送回预处理页
3. A 用户再继续点击灰度化、缩放、旋转、参数展示等按钮

这个按钮是对齐验收“现场采集一张脸并演示预处理过程”的关键。

## 7. 联调顺序

联调必须按顺序来，不要乱并。

### 阶段 1：主 GUI 空壳

完成条件：

1. `run_main.m` 可以打开窗口
2. 两个 tab 都存在
3. 所有按钮先能点，但逻辑可以是占位

### 阶段 2：预处理链路

完成条件：

1. 输入图像
2. 清除图像
3. 灰度化
4. 均衡化
5. 缩放
6. 旋转
7. 直方图
8. 获取图像参数

### 阶段 3：模型离线跑通

完成条件：

1. 训练脚本单独可跑
2. 单张预测单独可跑
3. 全量测试单独可跑
4. 平均脸和特征脸单独可生成

### 阶段 4：静态识别进 GUI

完成条件：

1. 单张识别按钮可跑
2. 全量识别按钮可跑
3. 结果表可显示

### 阶段 5：实时识别进 GUI

完成条件：

1. 摄像头可打开
2. 人脸可裁剪
3. 识别结果可刷新
4. 采集结果可送回预处理页复现

## 8. 每天的接口检查清单

每天晚上合并前，双方都要回答这几个问题：

1. `train_pca_svm_model` 的输入输出字段有没有改？
2. `predict_face_identity` 的返回字段有没有改？
3. `run_batch_test` 的结果表字段有没有改？
4. `replayPkg` 结构有没有改？
5. 默认图片尺寸、路径、标签规则有没有改？

如果改了，就必须同步更新这份文档和调用代码。

## 9. 最终统一入口

整个工程最终只保留一个启动方式：

```matlab
run_main
```

验收现场所有动作都从这个入口开始，不允许依赖多个散乱脚本。
