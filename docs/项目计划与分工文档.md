# 2026 线性代数人脸识别项目计划与分工文档

## 1. 项目目标

根据 `2026脸线性代数项目测试说明.pdf`，本项目验收重点是：

1. 识别精度，权重 60%。
2. GUI 可操作性，权重 20%。
3. 创新性小项目，权重 20%。

验收时必须通过 GUI 完成以下关键演示：

1. 测试集整体识别与单张识别。
2. 摄像头实时采集至少 10 位同学并完成识别。
3. 展示预处理过程：人脸检测、灰度变换、图像代数几何运算、人脸配准。
4. 展示 PCA 过程、超参数设置、平均脸、特征脸。
5. 展示 SVM 超参数设置。

本次方案明确改为：

1. 不使用现有项目代码。
2. 只保留现有数据集和项目要求。
3. 必须全程使用 MATLAB 完成。
4. 一周内从零重做一套识别率更稳、结构更清晰的新系统。

## 2. 为什么不复用现有项目

现有项目不作为交付基础，原因很明确：

1. 识别率差，继续修补不划算。
2. 旧代码历史包袱重，越修越乱的概率很高。
3. 你们只有一周，还主要依赖 Codex，最怕把时间浪费在旧 bug 上。

所以这次最合理的路线不是“继续补”，而是“砍掉重做，但只做验收必需功能”。

## 3. 新技术路线

项目要求只能使用 MATLAB，因此新的实现路线必须完全基于 MATLAB 重建。

建议技术方案：

1. GUI：MATLAB `uifigure` + `uibutton` + `uiaxes` + `uitabgroup`
2. 图像处理：MATLAB 图像处理基础函数
3. 摄像头采集：MATLAB 摄像头接口
4. PCA：自己实现或显式写出 PCA 过程
5. SVM：自己实现训练与预测过程，不能调用现成分类器库
6. 平均脸、特征脸展示：用矩阵转图像直接显示

建议不要继续使用旧 App Designer 文件，而是从零写新的 `.m` 文件程序。

原因如下：

1. 从零写新的 MATLAB 程序结构，比改旧 GUI 更干净。
2. 用脚本和函数搭 GUI，更容易拆模块，让两个人并行。
3. 一周内最需要的是结构清楚、接口清楚、方便 Codex 生成和重构。

## 4. 新项目核心方案

识别主链路建议固定为：

1. 输入图像或摄像头帧
2. 人脸检测
3. 关键点定位或人工辅助定位
4. 人脸校准
5. 统一裁剪到固定尺寸
6. 灰度化
7. 手写 PCA 降维
8. 手写 SVM 分类

这个方案的好处：

1. 符合课程主题，PCA 和 SVM 都能展示。
2. 识别率主要由“检测稳定 + 校准统一 + 训练规范”决定，比旧工程乱跑更稳。
3. 平均脸、特征脸、超参数展示都能自然挂进 GUI。

## 5. GUI 功能范围

### 5.1 界面一：图像预处理

左侧区域：

1. 输入图像
2. 清除图像
3. 灰度化
4. 均衡化
5. 启动摄像头
6. 停止摄像头

中间区域：

1. 显示当前输入图像
2. 摄像头模式下显示当前帧
3. 人脸捕获后显示框选结果
4. 人脸校准后显示校准结果

右侧区域：

1. 获取图像参数
2. 展示最大灰度值、最小灰度值、灰度均值、灰度方差
3. 绘制灰度直方图
4. 人脸捕获
5. 人脸校准
6. 图像缩放展示
7. 图像旋转

补充约束：

1. 图像旋转参数填角度。
2. 图像缩放参数填倍率，例如 `0.5`、`1.0`、`1.5`、`2.0`。
3. 人脸校准优先做“可展示、可微调”的版本，不做复杂编辑器。

### 5.2 界面二：人脸识别

左侧区域：

1. PCA 主成分数输入框
2. SVM 正则化参数输入框
3. 测试集全量识别按钮
4. 静态识别按钮
5. 显示特征脸按钮
6. 显示平均脸按钮
7. 实时识别按钮

右侧区域：

1. 静态识别结果展示区
2. 原始摄像头画面窗口
3. 送入识别器的人脸窗口
4. 识别出的人员信息文本区

静态识别流程固定为：

1. 点击测试集全量识别
2. 使用当前参数重新训练模型
3. 一次性读取测试集全部人脸
4. 显示总体准确率、总体耗时、各图片识别结果表
5. 点击静态识别
6. 选择单张测试集图片
7. 显示预测身份、候选结果、耗时

实时识别流程固定为：

1. 点击实时识别
2. 打开摄像头
3. 对当前帧做人脸检测与裁剪
4. 将裁剪后人脸送入识别器
5. 同时显示原始画面、识别人脸、识别名字、处理耗时
6. 提供“送入预处理页复现”按钮，把当前采集人脸送回预处理界面继续展示检测、灰度变换、几何运算和配准过程

## 6. 最合适的两人分工

### 6.1 结论

不要按“两个页面一人一个”分工，最合适的是按链路分工：

1. A 同学负责 `GUI + 预处理 + 摄像头 + 最终集成`
2. B 同学负责 `PCA/SVM 识别 + 训练评测 + 平均脸/特征脸产物`

这是最适合在 MATLAB 里从零重建并用 Codex 推进的方式，因为文件边界更清晰，冲突最少。

### 6.2 为什么不按页面分工

如果按页面分工，会有三个问题：

1. 识别页强依赖预处理页输出的人脸尺寸和格式。
2. 两个人都会反复改主 GUI 文件，最后很容易冲突。
3. 联调时参数、路径、返回结构容易对不上。

## 7. 两人协作规则

### 7.1 文件 ownership

1. `ui/` 和 `preprocess/` 目录默认由 A 同学主改。
2. `ml/` 目录默认由 B 同学主改。
3. B 同学不要直接大改 GUI 页面文件，只提供接口函数和调用说明。
4. A 同学只在接入阶段改识别按钮回调和展示逻辑。

### 7.2 先锁接口，再并行开发

第一天必须锁定以下接口：

1. 输入图像格式：RGB 还是灰度。
2. 识别器输入尺寸：例如 `112x92`。
3. 标签返回格式：例如姓名字符串。
4. 预测返回字段：预测类别、候选类别、耗时、人脸框。
5. 平均脸和特征脸的输出格式：矩阵还是图片路径。
6. 测试集全量识别结果表的字段。
7. 实时采集人脸回放到预处理页时的中间结果格式。

建议统一接口：

1. `model = train_pca_svm_model(trainDir, pcaDim, svmC, options)`
2. `result = predict_face_identity(model, imageData, options)`
3. `avgFace = build_average_face(model)`
4. `eigenfaceImgs = build_eigenface_preview(model, k)`
5. `batchResult = run_batch_test(model, testDir, options)`
6. `manifest = export_dataset_manifest(dataDir, trainDir, testDir, classSizeInfo)`

建议 `result` 至少包含这些字段：

1. `name`
2. `topKNames`
3. `topKScores`
4. `faceBox`
5. `alignedFace`
6. `elapsedMs`
7. `status`

建议 `batchResult` 至少包含这些字段：

1. `accuracy`
2. `totalElapsedMs`
3. `perImageResults`
4. `confusionSummary`
5. `status`

### 7.3 Git 协作方式

1. 主分支只保留可运行版本。
2. 两个人各自开分支开发新工程。
3. 每天晚上合并一次，不要拖到最后一天。
4. 合并前必须走一遍 GUI 主流程。

## 9. 一周内必须放弃的想法

为了保证重做后真的能交付，这些不要做太深：

1. 不要自己从零写复杂人脸检测器。
2. 不要做重型深度学习训练。
3. 不要做很复杂的拖拽校准系统。
4. 不要做花哨 UI 动效。
5. 不要一开始就支持太多图像变换种类。

一周内最值钱的是“识别率稳定 + GUI 不出错”，不是炫技。

## 10. 创新小项目

创新小项目不能最后临时口头补，必须作为正式交付项规划。

建议选择一个风险低、和主系统强相关的创新点：

1. 光照鲁棒模式：在识别前增加光照补偿开关，对比开启前后识别结果。
2. 多预处理策略对比：在 GUI 中切换“仅灰度化”和“灰度化+均衡化”，展示识别率差异。
3. Top-k 候选展示：不仅显示第一名，还显示前 3 名候选和得分。

推荐优先做第 2 项，因为实现成本最低，也最容易结合课程内容解释。

创新小项目负责人建议：

1. B 同学负责算法逻辑。
2. A 同学负责 GUI 开关和展示区。

创新小项目完成标准：

1. 能通过 GUI 单独触发。
2. 能展示结果差异或对比图。
3. 能在 PPT 中单独占 1 页。

## 12. PPT 中的任务分工写法建议

可以直接写：

### 方案一：50% / 50%

1. A 同学（50%）：GUI 设计与实现、图像预处理模块、摄像头模块、系统集成与演示联调。

## 13. 风险与应对

### 风险 1：GUI 功能太多，来不及

应对：

1. 先做验收必需功能。
2. 功能分基础版和增强版。
3. 人脸校准只做可展示版本。

### 风险 2：实时识别速度慢

应对：

1. 固定输入尺寸。
2. 实时阶段只做预测，不重新训练。
3. 必要时降低刷新频率，优先保稳定。

### 风险 3：两个人都改 GUI 导致冲突

应对：

1. GUI 页面只让 A 同学主改。
2. B 同学通过 `ml/` 目录独立提供接口。

### 风险 4：验收现场路径失效或摄像头打不开

应对：

1. 默认路径全部用相对路径。
2. 提前在验收机器测试摄像头权限。
3. 准备测试图兜底。

### 风险 5：从零重做时间不够

应对：

1. 第一天只搭骨架，不抠细节。
2. 第三天前必须先打通静态识别闭环。
3. 实时识别允许先做低帧率稳定版。
4. 创新点不要最后一天才开始。

## 14. 最终建议

这一周不要追求“做一个很全的大平台”，而要追求“从零重做后，验收老师点到哪一项，你们都能立刻通过 GUI 演示出来”。

优先级顺序应该是：

1. 静态识别准确率。
2. 测试集全量识别与结果表展示。
3. 实时识别可用性。
4. GUI 演示完整性。
5. 平均脸、特征脸、PCA/SVM 参数展示。
6. 创新点展示。

如果时间紧，宁可少做功能，也不要留下按钮能点但结果不稳定的东西。
